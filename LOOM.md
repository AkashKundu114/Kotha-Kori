## AI-SATHI Documentation

### Overview

AI-SATHI is a voice-first WhatsApp assistant for West Bengal SHG women. The system combines conversational support, pricing guidance, negotiation safeguards, and market-intelligence features with a strong emphasis on deterministic business logic, consent handling, and separation of product code from research data.

![generated-image-at-00:00:00](https://loom.com/i/bd38e36d11b74057b19ea6df50bfeb9c?workflows_screenshot=true)

- AI-SATHI is a WhatsApp-based, voice-first assistant for SHG women in West Bengal.
- Core capabilities include Bengali voice support, note keeping, pricing guidance, negotiation support, catalog posting/requisition, and market intelligence.
- The project is also tied to an in-progress paper, with feedback from real SHG participants informing product decisions.
- Consent forms and participant feedback are kept separate from the product code and used only for reporting and analysis.

**System Architecture** [0:50](https://loom.com/share/c0f2cd3f471b41ecad5a7486f00d43f5?t=50)

![generated-image-at-00:00:50](https://loom.com/i/ee8895b87c4642029ddb21131d845d1d?workflows_screenshot=true)

- The system is split into services with clear boundaries.
- **Gateway** owns the WhatsApp webhook boundary, including verification, deduplication, rate limiting, and async handoff.
- **Orchestrator** owns conversation state and feature routing.
- The orchestration approach is explicit: workflows are modeled as nodes and edges rather than scattered conditional logic.
- The codebase includes graph-based routing for people, intent, media, and compiler-related flows.

**Model Routing and Fallbacks** [1:16](https://loom.com/share/c0f2cd3f471b41ecad5a7486f00d43f5?t=76)

![generated-image-at-00:01:16](https://loom.com/i/3a0526c8590c40aaa944efe40c4aee26?workflows_screenshot=true)

- Model calls are centralized so strategy, timeouts, fallback behavior, and type handling are enforced in one place.
- The current stack prefers a primary model first, with local Ollama as a fallback when enabled.
- If no model is available, the system catches the failure and returns a friendly warning instead of crashing.
- There is no OpenAI dependency in the codebase.

**Pricing Flow** [1:48](https://loom.com/share/c0f2cd3f471b41ecad5a7486f00d43f5?t=108)

![generated-image-at-00:01:48](https://loom.com/i/b68ea4a22f0e4561bd5b445dc8e4aac8?workflows_screenshot=true)

- Pricing is handled deterministically because money-related decisions should not be generated by the model.
- The model is used only after the number is already computed, mainly to explain the result in friendly Bengali.
- The pricing mode includes an outbound message format that interprets the computed code rather than inventing the price.
- This keeps the business logic stable while still allowing natural-language responses.

**Negotiation Safety** [2:49](https://loom.com/share/c0f2cd3f471b41ecad5a7486f00d43f5?t=169)

![generated-image-at-00:02:49](https://loom.com/i/02cf0629e1814f599181c50f2f4d16c0?workflows_screenshot=true)

- Negotiation is treated as a high-risk flow because mistakes can cost sellers money.
- Invalid or undesirable amounts are rejected.
- Counter-offers are computed by pure logic and cannot go below the seller’s floor.
- The LLM is not allowed to write amounts directly; it can only provide a short reason.
- If the reason includes digits or number words, it is discarded.
- The guiding principle is that prompts may suggest behavior, but code enforces the rules.

**Negotiation Mode Flow** [3:20](https://loom.com/share/c0f2cd3f471b41ecad5a7486f00d43f5?t=200)

![generated-image-at-00:03:20](https://loom.com/i/f2f155f20cc0435f9a40922e69ba7960?workflows_screenshot=true)

- The negotiation flow includes instructions, number-word handling in Bengali, and structured reasoning steps.
- The flow covers starting negotiation, continuing negotiation, extracting an amount, evaluating an offer, and computing a counter-offer.
- There is also a counter-checking step to verify the result before proceeding.
- The implementation is designed to keep negotiation responses constrained and predictable.
